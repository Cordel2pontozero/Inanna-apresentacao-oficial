# Guia rápido de upload — Inanna

## Estrutura esperada

```text
inanna-apresentacao-oficial/
├── README.md
├── index.html
├── styles.css
├── script.js
├── manifest.webmanifest
├── robots.txt
├── sitemap.xml
├── LICENSE.md
├── .nojekyll
└── docs/
    ├── APRESENTACAO.md
    ├── ARQUITETURA.md
    ├── ROADMAP.md
    └── SEO_E_REPOSITORIO.md
```

## Como criar `docs/` no GitHub

Abra cada documento, clique no lápis e altere o nome para:

- `docs/APRESENTACAO.md`
- `docs/ARQUITETURA.md`
- `docs/ROADMAP.md`
- `docs/SEO_E_REPOSITORIO.md`

O GitHub cria a pasta automaticamente.

## Depois do upload

- configure descrição, website e tópicos;
- ative GitHub Pages para a vitrine;
- mantenha o aplicativo de produção na Vercel;
- teste a imagem oficial, o botão do aplicativo e o repositório técnico;
- não envie `.env`, tokens ou chaves.
