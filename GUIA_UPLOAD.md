# Guia rápido de upload — IZA

## Estrutura esperada

```text
iza-apresentacao-oficial/
├── README.md
├── index.html
├── styles.css
├── script.js
├── manifest.webmanifest
├── robots.txt
├── sitemap.xml
├── LICENSE.md
├── .nojekyll
└── docs/
    ├── APRESENTACAO.md
    ├── ARQUITETURA.md
    ├── ROADMAP.md
    └── SEO_E_REPOSITORIO.md
```

## Como criar `docs/` no GitHub

Abra cada documento, clique no lápis e altere o nome para:

- `docs/APRESENTACAO.md`
- `docs/ARQUITETURA.md`
- `docs/ROADMAP.md`
- `docs/SEO_E_REPOSITORIO.md`

O GitHub cria a pasta automaticamente.

## Depois do upload

- configure descrição, website e tópicos conforme `docs/SEO_E_REPOSITORIO.md`;
- ative GitHub Pages;
- confirme se as imagens oficiais carregam;
- teste a página em desktop e celular;
- mantenha o repositório técnico original separado desta vitrine.
